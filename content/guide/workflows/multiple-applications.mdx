# Managing Multiple Applications

Stay organized when applying to many jobs simultaneously.

## Organization Strategy

### Naming Convention

Use consistent version names:
```
[Company] - [Role] - [Date]
Acme Corp - Senior Dev - Jan2024
TechStart - Engineer - Jan2024
BigCo - Lead Developer - Jan2024
```

### Tracking System

Create a simple spreadsheet or note:

| Company | Role | Date Applied | Resume Version | Status |
|---------|------|--------------|----------------|--------|
| Acme Corp | Senior Dev | Jan 10 | Acme version | Interview |
| TechStart | Engineer | Jan 12 | TechStart version | Applied |

## Version Strategy

### Don't Over-Create

Common mistake: Creating too many versions

Better approach:
1. **Master Resume** — Your complete, general resume
2. **2-3 Industry Versions** — Tailored for different domains
3. **Company-Specific** — Only for your top choices

### When to Create New vs. Modify

**Create a new version when:**
- Targeting a completely different role type
- Company requires significantly different emphasis
- You want to preserve the original

**Modify existing version when:**
- Minor tweaks for similar roles
- Quick keyword additions
- Small customizations

## Batch Application Tips

### Efficient Workflow

1. **Group similar jobs** — Apply to similar roles together
2. **Use templates** — Start from industry versions
3. **Customize key sections** — Summary and top skills
4. **Quick review** — Check name/role accuracy

### Quality Over Quantity

Resist the urge to mass-apply with a generic resume.

Better results from:
- 10 customized applications
- Than 50 identical ones

## Staying Organized

### Weekly Review
- Check application statuses
- Follow up on pending applications
- Archive completed/rejected applications

### Keep Notes
For each application, note:
- Why you're excited about this role
- Key points to mention in interviews
- Questions you have for them
