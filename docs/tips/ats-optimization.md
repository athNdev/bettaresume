# ATS Optimization

Make sure your resume passes Applicant Tracking Systems.

## What is ATS?

Most companies use software to screen resumes before humans see them. These Applicant Tracking Systems (ATS) scan for:
- Keywords matching the job description
- Standard formatting
- Required qualifications

## ATS-Friendly Formatting

### Do's

- ✅ Use standard section headers (Experience, Education, Skills)
- ✅ Include keywords from the job posting
- ✅ Use simple, clean formatting
- ✅ Spell out acronyms (at least once)
- ✅ Use standard fonts

### Don'ts

- ❌ Use tables for layout
- ❌ Put important info in headers/footers
- ❌ Use images or graphics for text
- ❌ Use uncommon section titles
- ❌ Rely only on icons for information

## Keyword Strategy

### Finding the Right Keywords

1. Read the job posting carefully
2. Note repeated terms
3. Identify required skills and tools
4. Look at "required" vs "preferred" qualifications

### Where to Put Keywords

| Section | Strategy |
|---------|----------|
| Skills | List exact terms from posting |
| Experience | Use keywords in bullet points naturally |
| Summary | Include top 2-3 most important keywords |

### Natural Integration

**Bad:** "Python Java SQL AWS Docker Kubernetes"

**Good:** "Built data pipelines using Python and SQL, deployed to AWS with Docker containers"

## Section Headers

Use standard headers that ATS systems recognize:

| Use These | Not These |
|-----------|-----------|
| Experience | Career Journey |
| Education | Academic Background |
| Skills | What I Know |
| Summary | About Me |

## Testing Your Resume

### Manual Check

1. Copy your resume text to a plain text editor
2. Does it still make sense?
3. Is all information present?

### ATS Scoring Tips

- Match at least 60-70% of keywords
- Include both acronyms and full terms (e.g., "SEO (Search Engine Optimization)")
- Don't keyword-stuff—keep it readable

## Template Considerations

When choosing templates for ATS:

| Template Type | ATS Compatibility |
|---------------|-------------------|
| Simple/Clean | Best |
| Modern/Minimal | Good |
| Creative/Designed | May have issues |
| Heavy graphics | Avoid for ATS |

## Best Practices

1. **Customize for each application** — Keywords differ by posting
2. **Use the exact terms** — "Project Management" not "Managing Projects"
3. **Include hard skills** — Software, tools, methodologies
4. **Balance for humans** — ATS gets you past the gate, humans make decisions
